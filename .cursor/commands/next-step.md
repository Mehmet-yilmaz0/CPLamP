# next-step

When this command runs:

1. **Use the step-planner skill** — consider the implementation plan (docs/implementation_plan.md) and current repo state; only one concrete step per response.
2. Output **only**:
   - The **next single step** (what to do, which file/place — one clear action).
   - One short **reason** why this step comes now.

Do not list the full plan or future steps. One step + one reason; user does it, reports back, then you can run /next-step again for the following step.
